# Triage rubric

## Priority buckets
- **urgent**: security, legal, account access loss, payment failure, outage, or time-sensitive escalation
- **high**: needs attention today, user is blocked, or response should go out soon
- **normal**: important but not blocked, can wait until the next work block
- **low**: FYI, optional follow-up, or can safely wait

## Tie-breakers
- If the sender is blocked, raise priority.
- If a deadline is mentioned, raise priority.
- If there is a possible security or account risk, escalate.
- If the message is vague but clearly important, choose **high** and ask one clarifying question.

## Red flags
- payment or billing disputes
- account access or login issues
- security alerts or suspicious activity
- legal or compliance requests
- outages or broken production workflows
