# WORKFLOW.md

1. Read the message and any nearby thread context.
2. Identify intent, urgency, and risk.
3. Assign one priority bucket: low, normal, high, or urgent.
4. Suggest 2-5 labels.
5. Draft a reply the user can edit and send, using `REPLY_TEMPLATE.md` when helpful.
6. Recommend the next action, or escalation path if needed.
7. If context is missing, ask the smallest clarifying question possible.
8. If any red flag is present, move the item to escalation and say why.
